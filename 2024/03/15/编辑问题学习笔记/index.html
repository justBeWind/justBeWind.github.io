编辑问题学习笔记

前言

编辑距离问题，一般采用动态规划的解法，多为两个字符串之间，比如判读A是否为B的子序列，A和B之间，有多少种不同的子序列，删除A多少步才变为B，或者A编辑多少步才成为B…..

入门篇

判断子序列

题目链接：https://leetcode.cn/problems/is-subsequence/

给定字符串 s 和 t ，判断 s 是否为 t 的子序列。

字符串的一个子序列是原始字符串删除一些（也可以不删除）字符而不改变剩余字符相对位置形成的新字符串。（例如，”ace”是”abcde”的一个子序列，而”aec”不是）。

示例 1：

  • 输入：s = “abc”, t = “ahbgdc”
  • 输出：true

由于题目只考虑了字符串的删除情况，没有考虑增加和替换的情况，所以也属于编辑距离类型的题目，但是简化了很多操作。

动态规划解法：

dp[i][j] 表示以下标i-1为结尾的字符串s，和以下标j-1为结尾的字符串t，相同子序列的长度为dp[i][j]。

如果 最后我们发现 它们的公共子序列长度 和 s本身的长度 相等，那么s就一定是t的子序列，否则就不是。

特别注意，这里面为什么dp[i][j] 没有用常规的下标为i结尾的字符串s，和下标为j结尾的字符串t的含义

主要是这样的特殊定义，能简化初始化的步骤，以及递归中的处理

有了dp数组的含义后，那么我们要考虑初始化

对于dp[0][j] 显然公共子序列的长度为0

对于dp[i][0] 显然公共子序列的长度也为0

接下来要确定递推公式：

  • if (s[i - 1] == t[j - 1])
    • t中找到了一个字符在s中也出现了
  • if (s[i - 1] != t[j - 1])
    • 相当于t要删除元素，继续匹配

if (s[i - 1] == t[j - 1])，那么dp[i][j] = dp[i - 1][j - 1] + 1;，因为找到了一个相同的字符，相同子序列长度自然要在dp[i-1][j-1]的基础上加1

if (s[i - 1] != t[j - 1])，此时相当于t要删除元素，t如果把当前元素t[j - 1]删除，那么dp[i][j] 的数值就是 看s[i - 1]与 t[j - 2]的比较结果了，即：dp[i][j] = dp[i][j - 1];

不同的子序列

题目链接：https://leetcode.cn/problems/distinct-subsequences/description/

给定一个字符串 s 和一个字符串 t ，计算在 s 的子序列中 t 出现的个数。

字符串的一个 子序列 是指，通过删除一些（也可以不删除）字符且不干扰剩余字符相对位置所组成的新字符串。（例如，”ACE” 是 “ABCDE” 的一个子序列，而 “AEC” 不是）

题目数据保证答案符合 32 位带符号整数范围。

示例 1：

由于题目考虑的是t在s的子序列中出现的次数，即我们要对s进行删除操作，也可以不删除来得到t，最终看有多少个方案，所以也属于编辑距离问题。

首先还是老四步，确定dp数组含义，初始化，以及递推公式和遍历顺序

1、确定dp数组含义

dp[i][j]：以i-1为结尾的s子序列中出现以j-1为结尾的t的个数为dp[i][j]。

这里用特殊含义的dp数组也是为了方便初始化和遍历过程中的处理

2、初始化

单独考虑s 和 t分别为0的情况

  • ​ 当s为0时候 那么dp[0][j] 都是0，
  • ​ 当t为0时候，那么dp[i][0] 都是1

3、确定递推公式（动态转移方程）

对于子序列，我们要分析两种情况

  • s[i - 1] 与 t[j - 1]相等
  • s[i - 1] 与 t[j - 1] 不相等

当s[i - 1] 与 t[j - 1]相等时，s中构成t的方案可以有两种可能：

一使用这个条件，利用s[i - 1] == t[j - 1]，看s[0…i-2] 和 t[0…j-2]的情况。对应dp[i-1][j-1]

二不使用这个条件，可能s[0…i-2] 和 t[0…j-1] 相等。 对应dp[i-1][j]

如果s[i - 1] 与 t[j - 1]不相等，那么s[i-1] 就不用管了，相当于把它删除掉，就看s[0…i-2] 和t[0…j-1]的情况

4、遍历顺序

从递推公式和初始化可以发现，后面的状态依赖于前面的状态，所以i和j都是从前往后遍历。

提高篇

两个字符串的删除操作

题目链接：https://leetcode.cn/problems/delete-operation-for-two-strings/description/

给定两个单词 word1 和 word2，找到使得 word1 和 word2 相同所需的最小步数，每步可以删除任意一个字符串中的一个字符。

示例：

  • 输入: “sea”, “eat”
  • 输出: 2
  • 解释: 第一步将”sea”变为”ea”，第二步将”eat”变为”ea”

这道题目 和上一道不同的子序列相比，改变在于 现在两个字符串都可以删除了。

还是老四步，确定dp数组含义，初始化，以及递推公式和遍历顺序

1、确定dp数组含义

dp[i][j]：以i-1为结尾的字符串word1，和以j-1位结尾的字符串word2，想要达到相等，所需要删除元素的最少次数。

dp数组的特殊定义是为了方便初始化

2、初始化

dp[i][0] 和 dp[0][j]是一定要初始化的。

dp[i][0]：word2为空字符串，以i-1为结尾的字符串word1要删除多少个元素，才能和word2相同呢，很明显dp[i][0] = i。

dp[0][j]的话同理

3、确定递推公式

  • 当word1[i - 1] 与 word2[j - 1]相同的时候，dp[i][j] = dp[i - 1][j - 1];
  • 当word1[i - 1] 与 word2[j - 1]不相同的时候

如果不相同的时候，由于两个字符串都能删除，就会出现三种分类讨论的情况：

一：删word1[i - 1]，最少操作次数为dp[i - 1][j] + 1

二：删word2[j - 1]，最少操作次数为dp[i][j - 1] + 1

三：同时删word1[i - 1]和word2[j - 1]，操作的最少次数为dp[i - 1][j - 1] + 2

由于我们要求的是最小值，所以当不相同的时候，递推公式就取这三种情况下的最小。

4、遍历顺序

从递推公式中可以看出，后面的值依赖于前面的，所以从前向后遍历。

编辑距离

题目链接：https://leetcode.cn/problems/edit-distance/

给你两个单词 word1 和 word2，请你计算出将 word1 转换成 word2 所使用的最少操作数 。

你可以对一个单词进行如下三种操作：

  • 插入一个字符
  • 删除一个字符
  • 替换一个字符
  • 示例 1：
  • 输入：word1 = “horse”, word2 = “ros”
  • 输出：3
  • 解释： horse -> rorse (将 ‘h’ 替换为 ‘r’) rorse -> rose (删除 ‘r’) rose -> ros (删除 ‘e’)

还是老四步，确定dp数组含义，初始化，以及递推公式和遍历顺序

1、确定dp数组含义

dp[i][j] 表示以下标i-1为结尾的字符串word1，和以下标j-1为结尾的字符串word2，最近编辑距离为dp[i][j]。

2、初始化

dp[i][0] ：以下标i-1为结尾的字符串word1，和空字符串word2，最近编辑距离为dp[i][0]。

那么dp[i][0]就应该是i，对word1里的元素全部做删除操作，即：dp[i][0] = i;

同理dp[0][j] = j;

3、递推公式（动态转移方程）

对于不等的情况，

  • 操作一：word1删除一个元素，那么就是以下标i - 2为结尾的word1 与 j-1为结尾的word2的最近编辑距离 再加上一个操作。

即 dp[i][j] = dp[i - 1][j] + 1;

  • 操作二：word2删除一个元素，那么就是以下标i - 1为结尾的word1 与 j-2为结尾的word2的最近编辑距离 再加上一个操作。

即 dp[i][j] = dp[i][j - 1] + 1;

特别注意，删除元素和添加元素是等效的，比如说word1删除一个元素，相当于word2添加一个元素，它们的操作数是一样的。

操作三：替换元素，word1替换word1[i - 1]，使其与word2[j - 1]相同，此时不用增删加元素。

只需要一次替换的操作，就可以让 word1[i - 1] 和 word2[j - 1] 相同。

所以 dp[i][j] = dp[i - 1][j - 1] + 1;

4、遍历顺序

从递推公式可以发现，遍历的顺序还是要从前往后
